💡 Fluxo Principal

Nova conversa llega → Bot executa fluxo automaticamente → Se nenhuma ação terminal ocorrer, conversa entra na fila para agente

Cenários Possíveis

Cenário 1: Fluxo Resolve (Cliente vai para Fila)

1

Cliente envia mensagem

Ex: "Oi, preciso de ajuda"

2

Bot envia fluxo: Boas-vindas

"Olá! Qual o tipo de atendimento?"

3

Cliente escolhe opção (1, 2 ou 3)

Seleção registrada na conversa

4

Bot envia mensagem de confirmação

"Ótimo! Um agente irá ajudar..."

5

Conversa entra em FILA com contexto

Agente vê: "Cliente escolheu opção 1", histórico do bot, etc

Cenário 2: Fluxo Resolve Sozinho (Sem entrar na Fila)

1

Cliente envia mensagem

Ex: "Qual seu horário de funcionamento?"

2

Bot reconhece e responde (com condicional)

Resposta automática: "Funcionamos de 8h às 18h"

3

Bot pergunta se resolveu

"Sua dúvida foi resolvida? (Sim/Não)"

4

Se responder "Sim": Encerra conversa

Status: resolved. NÃO entra na fila

5

Se responder "Não": Transfere para agente

Entra na fila com contexto

Configuração do Sistema

✅ Banco de Dados

Criar tabela conversation_flows:

id, name, trigger_type (on_new_conversation), enabled (true/false), nodes (JSON), created_by, created_at, updated_at

✅ Acionamento Automático

Após criar a conversa no WhatsAppService, verificar:

if (conversation.status === 'new' && flowEnabled) { flowService.executeFlow(conversation, flowId) }

⚠️ Importante

O fluxo é "automático" mas responde às mensagens do cliente. Ou seja:

  • Cliente envia msg 1 → Bot responde com bloco 1
  • Cliente envia msg 2 (resposta à escolha) → Bot processa e vai para bloco 2
  • Conversa continua até terminar o fluxo ou transferir
Fluxo no Código

📍 Onde integra

No webhook (WhatsAppService.php):

1. Webhook chega → cria/encontra conversation 2. Verifica: exists(flow_by_trigger='on_new_conversation')? 3. SIM → FlowService::executeFlow($conversation, $flow) 4. Fluxo executa (pode enviar N mensagens) 5. Se flow termina com "transfer" → status fica 'new', entra fila 6. Se flow termina com "resolve" → status 'resolved', encerra
